Sintáxis de un elemento

Valores(values): siempre encerrados entre comillas, y seguidos de un signo igual, especifican las configuraciones establecidas por el atributo dle elemento HTML.
visitanos
Atributos(attributes): son parámetros adicionales en los elementos HTML, que modifican su funcionamiento y presentación de multiples maneras.
- src en img para especificar la URL de una imagen.
- alt en img para proporcionar texto alternativo en caso de que la imagen no se pueda mostrar.
- class para asignar una o más clases a un elemento, lo que es útil para aplicar estilos CSS.
- id para asignar un identificador único a un elemento, útil tanto para estilos como para manipulación con JavaScript.
- style para aplicar estilos CSS directamente a un elemento.

Elementos en línea vs. Elementos en bloque

Elementos en bloque: Estos elementos generalmente comienzan en una nueva línea y ocupan todo el ancho disponible. Forman un "bloque" visible en la página.
Ejemplos: div(es un elemento en bloque que agrupa todo el contenido de la sección del artículo), p(son elementos en bloque para párrafos, cada uno ocupando su propia línea en la disposición.), h1(es un elemento en bloque que sirve como el título principal del artículo y comienza en una nueva línea.), y form.
Son útiles para estructurar el layout y agrupar otros elementos.
Elementos en línea: Estos elementos no comienzan en una nueva línea y solo ocupan tanto ancho como su contenido lo requiera.
Ejemplos: span, a( es un elemento en línea que se utiliza dentro de un párrafo para crear un enlace sin empezar en una nueva línea.), img, y b.
Son adecuados para usar dentro de los elementos en bloque cuando quieres manipular solo una parte del texto o insertar pequeños componentes como imágenes.
Cada tipo de elemento tiene su propósito específico en la estructura y presentación del contenido web, y entender sus diferencias es clave para un diseño web efectivo.

El Cambio Climático

El cambio climático se refiere a un cambio significativo en las medidas del clima que dura por un período extendido de tiempo.

El término "layout" se refiere a la disposición y estructura de elementos dentro de una página web. Es la forma en que se organizan los contenidos, imágenes, formularios y otros elementos en el navegador. Un buen layout debe ser estéticamente agradable y funcional, guiando al usuario de manera intuitiva a través del flujo de información y acciones disponibles en el sitio. En el desarrollo web, trabajar en el layout implica tomar decisiones sobre la disposición de los bloques de contenido, la alineación, el espaciado, y cómo estos se adaptarán o responderán a diferentes tamaños de pantalla y dispositivos.

Listas

Listas ordenadas:utilizan la etiqueta ol y presentan elementos en una secuencia numérica o alfabética. Cada elemento de la lista se marca con la etiqueta li. Son útiles cuando el orden de los ítems es importante, como en recetas o instrucciones.
  1. Preparar los ingredientes.
  2. Mezclar los ingredientes secos.
 Listas desordenadas:se crean con la etiqueta ul y los elementos de la lista no tienen un orden específico. Esta es la lista que usarías para cosas como listas de compras o características.
  • Leche
  • Huevos
  • Harina
 Listas de definición:aunque no son tan comúnmente utilizadas, se emplean para presentar un término y su definición correspondiente. Se crean con la etiqueta d1 y dentro se colocan pares de términos (dt) y definiciones (dd).

Tablas

Son una forma de organizar datos en filas y columnas. Son muy útiles para presentar información de manera estructurada y fácil de leer.
Estructura básica
  • table: define una tabla
  • tr: define una fila de la table
  • th: define una selda de encabezado(normalmente en negrita)
  • td: define una celda de datos regular
  • thead: define el encabezado de la table
  • tbody: define el cuerpo de la table
  • tfoot: define el pie de la tabla
  • colspan: permite que una celda se extienda a través de múltiples columnas
  • rowspan: permite que una celda se extienda a través de múltiples filas
